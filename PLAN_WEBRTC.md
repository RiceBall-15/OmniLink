# WebRTC 音视频通话 - 并发开发计划

## 🎯 目标
在 OmniLink 中快速实现音视频通话功能，采用并发开发模式，最大化开发效率。

## 📋 技术栈
- **前端**：React + TypeScript + simple-peer (WebRTC封装)
- **信令**：WebSocket (socket.io-client)
- **后端接口**：JSON规范定义
- **风格**：快速原型，优先功能，性能优化后续

## 🔄 并发任务分配

### 任务组 1：类型定义（立即执行）
- [x] 定义 WebRTC 相关类型
- [x] 定义通话状态枚举
- [x] 定义信令消息类型

### 任务组 2：核心 Hooks（并发执行）
- [ ] useWebRTC - WebRTC 连接管理
- [ ] useMediaStream - 媒体流获取
- [ ] useCall - 通话状态管理

### 任务组 3：UI 组件（并发执行）
- [ ] VideoCall - 视频通话界面
- [ ] AudioCall - 语音通话界面
- [ ] CallControls - 通话控制按钮
- [ ] IncomingCall - 来电提示

### 任务组 4：后端接口定义
- [ ] 信令服务接口规范
- [ ] STUN/TURN 服务器配置
- [ ] 媒体流路由设计

## 🚀 开发原则
1. **不阻塞**：前端开发不等待后端实现
2. **用现成库**：simple-peer, socket.io
3. **快速原型**：功能优先，样式优化后续
4. **跳过测试**：手动验证即可，自动化测试后续
5. **并发执行**：4个子代理同时工作

## 📊 优先级
1. 🔥 P0 - 视频通话（1:1）
2. 🔥 P0 - 语音通话（1:1）
3. 🟡 P1 - 屏幕共享
4. 🟢 P2 - 群组通话

## ⏱️ 预期时间
- 任务组 1：5分钟
- 任务组 2-4（并发）：20分钟
- 总计：25分钟完成核心功能

## 🎨 UI 要求
- 简洁现代
- 渐变色设计
- 响应式布局
- 移动端适配

## 📝 注意事项
- 所有组件必须有完整的TypeScript类型
- 错误处理要完善（网络、权限、设备）
- 使用 useMemo/useCallback 优化性能
- WebSocket 自动重连
